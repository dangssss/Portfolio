Banking churn demo with SHAP and MAPIE.

1. Business Goal

The demo predicts which customers are likely to leave so a retention team can prioritize outreach. The important output is not only a churn label, but also why the customer is risky and whether the prediction is confident enough for action.

  • Use churn probability to rank customers for retention action.
  • Explain risk drivers with customer-level SHAP values.
  • Use MAPIE to separate clear predictions from ambiguous cases.

2. Data and Split

Raw fields

  • CreditScore, Gender, Age, Tenure
  • Balance, NumOfProducts, HasCrCard
  • IsActiveMember, EstimatedSalary
  • LoginFrequency, ExternalTransfers

Validation setup

  • 70% training set
  • 10% calibration set for MAPIE
  • 20% held-out test set
  • No missing values in the exported notebook run

5. Explanation Layer

SHAP turns the classifier into a business-facing explanation workflow. Global plots identify major churn drivers, while local waterfall plots explain why one customer is high risk.

  • ExternalTransfers is the strongest signal in the feature relevance output.
  • Ratio features help explain whether a customer is active relative to tenure and product ownership.
  • Customer-level explanations make the risk score actionable for retention discussions.

6. MAPIE Confidence Layer

MAPIE is fitted with the selected LightGBM model using a calibration set and alpha = 0.05. The output is a conformal prediction set rather than only one hard label.
