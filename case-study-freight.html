Supply Chain Shipment Type Prediction

Medical supply chain ML case focused on predicting shipment mode. Freight and weight prediction are supporting repair steps, while the main deliverable is a multiclass shipment-mode classifier with uncertainty-aware review routing.

1. Project Context

The broader research topic is AI for healthcare supply chain decision support. My part is the ML pipeline for shipment mode prediction, not the LLM or product-interface sections of the larger project.

  • Main question: Given historical shipment fields, which shipment mode should be recommended?
  • Target classes: Air, Truck, Air Charter, and Ocean.
  • Boundary: Freight quote prediction is an auxiliary data-repair step, not the final case-study title.

2. Data Understanding

Shipment fields

  • Project Code, Country, Vendor
  • Product Group, Molecule/Test Type
  • Line Item Quantity and Line Item Value
  • Pack Price, Unit Price, insurance cost

Operational fields

  • Delivery and recorded dates
  • Days-to-deliver derived signal
  • Weight (Kilograms)
  • Freight Cost (USD)

The notebook output shows 10,324 rows after loading and 9,964 rows after removing records without Shipment Mode. The remaining labels are used for supervised multiclass classification.

  • Converted Weight (Kilograms) and Freight Cost (USD) to numeric, then log-transformed targets with log1p.
  • Used RobustScaler for commercial numeric fields and added interaction features such as Weight_Quantity, Weight_Price, and Freight_Quantity.
  • Saved the XGBoost repair model, predicted missing rows, inverse-transformed predictions, and used the filled dataset for the final classifier.

7. MAPIE Review Gate

MAPIE is added on top of XGBoost with alpha = 0.05. The result is a prediction set that supports operational routing: auto-recommend clear cases and send ambiguous shipments to manual review.

Prediction setInterpretationDecision

8. Case Study Positioning

This page should present the project as shipment mode decision support. The strongest story is: repair incomplete operational data, train a strong multiclass model, and expose uncertainty so the system knows when not to over-automate.
